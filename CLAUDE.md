## このアプリは何か

学生ラグビーアナリスト向けの、スタッツ記録ツール（iOS）。

中心思想は「記録の詳細度を状況に合わせて選べる」こと。
時間がなければ得点だけ記録し、余裕があれば得点者まで記録する。
記録したものはすべて後から追記・修正できる。
「簡易か詳細か」の二択ではなく「段階的に詳細化できる」が本質。

設計判断に迷ったら常にこの基準で決める:
すべての情報は「必須ではなく、後から足せる」前提で作る。

ラグビーの形式（セブンズ／15人制、登録人数 12/13/15/23 等）は
アプリ側で固定しない。形式を決め打ちしないことが最も柔軟で、
かつ最も実装がシンプル。アプリは「何人制か」を知らなくてよい。

## データ構造（最初からこの形で作る・V1の土台）

6エンティティ。後から機能を足せるよう、この形を崩さない。

- Team: id, name
  自チームも相手チームも対等にこの1つで扱う。
- Player: id, team_id, number, name(任意=null可)
  選手はチームに属す。名前任意、未登録なら背番号のみで運用。
  同一チーム内で背番号(number)は一意。
- Tournament: id, official_name
  シーズンは持たない。正式名称1本。
- Match: id, tournament_id, home_team_id, away_team_id, played_at
  試合は2チームを参照（自チーム固定にしない）。
- StatEvent: id, match_id, player_id(任意=null可),
    category, outcome, seconds
  category 例: possession / try / conversion / penalty_goal /
    drop_goal / lineout / scrum
  outcome 例: success / fail、または保持側 own / opponent
  player_id は任意。得点を選手未指定でも保存でき後から紐づけ可。
  1テーブルで全種類のボタンを受ける。新項目は category 追加で対応。
- Substitution: id, match_id, player_in, player_out, minute
  **V1では使わない。構造だけ定義しておく。** V2で交代・出場
  時間の画面をこの上に乗せる。

設計の背骨は2つの任意フィールド:
Player.name 任意 → 背番号だけで記録開始できる。
StatEvent.player_id 任意 → 選手不明でも記録し後から紐づけ。

## バージョン1の範囲（これだけ作る）

画面は5つ。

### 1. 試合一覧（起動時のルート画面）
- 試合日の新しい順。各行に対戦カード・大会名・日付・状態。
- 状態は「記録中」「終了」。タップ時、記録中なら記録画面の続きへ、
  終了ならサマリーへ遷移（状態で遷移先が変わる）。
- 絞り込みはV1では持たない。オンボーディング画面も作らない。

### 2. 試合をつくる
- 大会選択(or この画面内で新規作成)、2チーム、日付。
- 自チーム名は設定のデフォルト値を表示（毎回入力させない）。
- 大会は正式名称のみ。大会作成専用画面は作らない。
- 試合日の初期値は「前回入力した試合日」。変更したらその値が
  次回の初期値（過去試合をまとめて記録する想定）。
- 保存ボタンは「保存」と「保存して記録開始」の2つ。

### 3. メンバー表登録（チーム編集画面内のセクション）
- 独立した巨大画面は作らない。
- 背番号1〜15の空欄を初期表示。超える分は「＋追加」で何番でも
  足せる。名前は任意で後から埋められる。
- 自チームも相手も同じ画面・同じ仕組みを使い回す。
- 背番号は同一チーム内で一意。
- 選手の完全削除はV1では持たない。

### 4. 記録画面（アプリの心臓部）
- ポゼッション: トグル型。タップで保持側切替。実時間で計測。
  切替時にその区間を1件のStatEvent(possession, 保持側, 区間秒数)
  として保存。「試合終了」時に計測中の最後の区間も必ず保存して
  締める。状態は own / opponent の2状態のみ。最初のポゼッション
  タップまで計測を始めない。
- 得点(try/conversion/penalty_goal/drop_goal): カウント型。
  タップで即StatEvent保存(選手未指定で確定)。同時に選手選択
  シートが下から出る。選べば紐づく、閉じれば選手なし。トライと
  コンバージョンは別StatEvent、選手が違ってよい、どちらも任意。
- セットプレー(lineout/scrum): 成否ペア型。「成功」「失敗」を
  別々に記録、成功率は表示時に計算。
- 「取り消し」を必ず置く。直前の1件のみ取り消し（履歴を遡らない）。
  取り消し挙動はカテゴリで異なる（ポゼッションは直前保存区間を
  消し保持状態を戻す。得点・セットプレーは直前イベントを消す）。
- セブンズ調整: 上部の経過時間は実時間で分:秒表示。前半/後半は
  アプリが時計管理しない。手動トグルを1つ置くだけ。ハーフは
  StatEvent に持たない（必要ならV2で half カラム追加。構造不変）。

### 5. 試合サマリー
- 3ブロック: ポゼッション(own対opponentの%と横棒1本)、得点
  (種類別回数と合計得点の表)、セットプレー(成功数/試行数と成功率
  の表)。
- 加えて得点者の時系列一覧(時間・種類・選手名)。選手未指定は
  「未設定」と表示（後から埋めるべき箇所が一目で分かる）。
- 数値は保存値でなく常に StatEvent から再計算（後から記録を
  足しても自動で正しくなる）。
- 得点者一覧の各行タップで選手選択シート（記録時と同一
  コンポーネントを使い回す。別々に作らない）。
- 得点自体の削除はこの一覧からスワイプ。試合自体の削除はV1なし。
- サマリーから記録画面に戻る導線を置く。

## 設定画面
V1で持つのはデフォルト自チーム名の1項目だけ。他は持たない。

## データのバックアップ
V1ではバックアップ機能を作らない。各レコードがUUIDを持つ構造
（既にそう）で将来同期の道だけ残す。「大事な大会の後は端末ごと
iCloudバックアップ推奨」の一文をアプリ内に出すに留める。本格的な
バックアップ/共有はV2最優先。
